# Data

This directory stores dataset splits and annotations used in the thesis. Raw image data (nuScenes) is **not committed to git** — download separately following the instructions below.

---

## Directory Structure

```
data/
├── drivelm/                      # Original DriveLM-nuScenes annotations
├── drivelm_custom_split/         # Custom i.i.d. 80/10/10 split (train/val/test)
├── drivelm_aug_pseudo_labels/    # Augmented training sets mixing DriveLM + pseudo-labels
└── nuScenes/                     # Symlink or path to nuScenes samples (not in git)
```

## 2. DriveLM-nuScenes (`drivelm/`)

Original DriveLM QA annotations. Download from the [DriveLM challenge repository](https://github.com/OpenDriveLab/DriveLM/tree/main/challenge):

| File | Description |
|---|---|
| `v1_1_train_nus.json` | Full training set with GT answers |
| `v1_1_val_nus_q_only_converted_llama.json` | Validation set — questions only, no GT answers (challenge submission format) |
| `train_sample.json` | Small training sample for quick development/demo |

### Preparing the data for training

The original DriveLM JSON must be extracted and converted before it can be used for training or inference. Follow the steps below (all commands assume you are under `./challenge` in the DriveLM repo):

**Step 1 — Extract QA pairs** from the raw training JSON into a flat `test.json`:
```bash
python extract_data.py
```

**Step 2 — Convert** to evaluation format `test_eval.json`:
```bash
python convert_data.py
```

**Step 3 — Convert to LLaMA-Adapter / InternVL format** (`test_llama.json`):
```bash
python convert2llama.py
```

`test_eval.json` is used for evaluation. `test_llama.json` (converted LLaMA format) is the input format used for training and inference in this thesis.

> For nuScenes image setup, either symlink your existing nuScenes directory or download the DriveLM nuScenes image subset from [Google Drive / HuggingFace / Baidu Netdisk](https://github.com/OpenDriveLab/DriveLM/tree/main/challenge) and unzip into `data/nuscenes/`. Expected structure:
> ```
> data/nuscenes/samples/
>   CAM_FRONT/
>   CAM_FRONT_LEFT/
>   CAM_FRONT_RIGHT/
>   CAM_BACK/
>   CAM_BACK_LEFT/
>   CAM_BACK_RIGHT/
> ```

## 3. Custom i.i.d. Split (`drivelm_custom_split/`)

The official DriveLM split is **not i.i.d.** — train and test scenes overlap temporally. This thesis constructs a clean **80/10/10 scene-level split** (Section 4.3 of the thesis) where no scene appears in more than one partition.

| File | Records | Description |
|---|---:|---|
| `train.jsonl` | 25,825 | Training QA pairs (80% of scenes) |
| `val.jsonl` | 2,989 | Validation split — used for checkpoint selection |
| `test.jsonl` | 3,340 | Test split — all evaluation results in this repo are reported on this set |

Each JSONL record contains: `id`, `image` (6 camera paths), `conversations`, `category`, `metric_type`, and template metadata.

To regenerate the split from the original DriveLM annotations:
```bash
bash scripts/01_build_custom_split.sh
```

## 4. Augmented Training Sets (`drivelm_aug_pseudo_labels/`)

Full dataset hosted on HuggingFace: **[huggingface.co/datasets/dkhursen/drivelm-pseudo-labels](https://huggingface.co/datasets/dkhursen/drivelm-pseudo-labels)**
> Large files (`drivelm_pseudo_qas_100k.json`, `train_aug30/50/100pct.json`) are excluded from git — download from HuggingFace. Only `train_aug10pct.json` (~52 MB) is kept locally as a sample.

Training sets that mix the custom split `train.jsonl` with pseudo-labels generated by the Qwen3-based pipeline (Chapter 5 of the thesis).

| File | Description |
|---|---|
| `drivelm_pseudo_qas_100k.json` | ~100k QA pairs generated by Qwen3 with chain-of-thought reasoning from nuScenes sensor priors |
| `train_aug10pct.json` | `train.jsonl` + 10% of pseudo-labels (best performing mix — see results) |
| `train_aug30pct.json` | `train.jsonl` + 30% of pseudo-labels |
| `train_aug50pct.json` | `train.jsonl` + 50% of pseudo-labels |
| `train_aug100pct.json` | `train.jsonl` + 100% of pseudo-labels |

The pseudo-labels were generated using the pipeline proposed in this thesis: structured sensor priors (3D bounding boxes, LiDAR depth, tracking trajectories) are passed to **Qwen3** with thinking enabled, which produces natural-language VQA pairs grounded in the physical annotations.

To regenerate pseudo-labels from scratch:
```bash
bash scripts/03_run_pipeline_nuscenes.sh
```

## 5. Valeo Dataset

Proprietary urban driving data collected by Valeo in Prague. **Not publicly available yet** — a public release is planned for the future. Until then, neither the raw data nor any models trained on Valeo data can be released due to company policy.

Results involving Valeo-trained models are reported in the thesis but excluded from this repository.
